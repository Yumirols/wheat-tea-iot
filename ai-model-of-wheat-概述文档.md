# 小麦病虫害 AI 检测模型 — 模块概述文档

## 1. 概述

本模块基于 **YOLOv8** 深度学习目标检测框架，构建了一套完整的小麦病虫害智能检测系统。系统涵盖 **模型训练**、**静态图像推理** 与 **实时摄像头检测** 三个核心环节，能够自动识别和定位小麦植株上的病虫害区域。

整个工作流以 Jupyter Notebook（`.ipynb`）为载体，分为三个独立可运行的 Cell 组：

| 阶段 | 功能 | 运行环境 |
|------|------|----------|
| 模型训练 | 使用标注好的小麦病虫害数据集训练 YOLOv8n 检测模型，并导出 ONNX | GPU（推荐）/ CPU |
| 图像检测 | 加载训练好的权重，对单张/多张静态图片进行推理与可视化 | CPU / GPU |
| 摄像头检测 | 调用本地摄像头，实现实时视频流的小麦病虫害检测 | 本地摄像头 + GPU（推荐） |

## 4. 设计要点

### 4.1 路径管理的健壮性

- 通过 `ROOT_DIR` 全局常量固定项目根目录，配合 `os.chdir()` 统一工作路径。
- 自定义 `get_abs_path()` 函数基于根目录拼接绝对路径，**彻底规避 Jupyter 中 `__file__` 不可用的报错**。
- 所有文件访问前调用 `check_file_exists()` 做前置校验，失败即退出，避免训练中途崩溃。

### 4.2 GPU 内存监控

- 训练前后调用 `print_gpu_memory_info()` 打印 CUDA 显存状态（已分配 / 已预留 / 峰值），便于 OOM 排查。
- 训练结束后主动 `torch.cuda.empty_cache()` 释放缓存。

### 4.3 数据增强策略

训练阶段配置了较丰富的在线数据增强，提升模型泛化能力：

| 参数 | 值 | 含义 |
|------|-----|------|
| `degrees` | 60 | 随机旋转 ±60° |
| `fliplr` | 0.5 | 50% 概率水平翻转 |
| `flipud` | 0.3 | 30% 概率垂直翻转 |
| `scale` | 0.6 | 缩放系数 0.6 |
| `translate` | 0.15 | 平移 15% |
| `perspective` | 0.001 | 透视变换 |

### 4.4 训练配置

- **基础模型**：YOLOv8n（nano 版本，轻量化）
- **训练轮数**：50 epochs
- **Batch Size**：16
- **输入分辨率**：640×640
- **早停机制**：`patience=10`（10 轮无改善则停止）
- **混合精度**：AMP 自动启用（有 GPU 时）
- **缓存策略**：`cache='ram'` 将数据预加载到内存加速训练
- **确定性**：固定随机种子 `torch.manual_seed(42)` 保证可复现

### 4.5 多模态推理支持

Notebook 支持三种推理模式：

1. **静态图像**：加载 `best.pt` → `model(img_path)` → Matplotlib 并排展示多图结果。
2. **实时摄像头**：`cv2.VideoCapture(0)` 捕获帧 → YOLO 推理 → IPython `clear_output` 实现 Jupyter 内流畅视频流显示。
3. **部署导出**：`model.export(format='onnx', simplify=True)` 导出精简 ONNX，便于生产环境跨平台部署。

### 4.6 中文显示兼容

- 通过 `plt.rcParams['font.sans-serif'] = ['SimHei']` 配置黑体。
- `axes.unicode_minus = False` 避免负号显示为方块。
- 图像标题、打印日志均使用中文。

### 4.7 代码容错与防御性设计

- 摄像头打开失败 → 打印提示并 `exit()`。
- 数据集配置文件缺失 → 打印路径并 `sys.exit(1)`。
- 训练异常 → `try/except` 捕获并截断告警信息（前150字符，避免刷屏）。
- 摄像头资源 → `try/finally` 确保 `cap.release()` 一定执行。

## 6. 注意事项与改进建议

1. **路径硬编码**：当前 `ROOT_DIR` 和测试图片路径均为硬编码的绝对路径（`D:\微信QQ\FireDetection`），迁移到其他机器需手动修改。建议改为相对路径或命令行参数。
2. **数据集未随附**：`datasets/data.yaml` 与标注图像需自行准备，Notebook 仅包含训练与推理代码。
3. **模型权重路径异常**：Cell 2 中的模型路径 `runs\detect\runs\detect\xiaomai_model\weights\best.pt` 存在重复的 `runs\detect` 层级，疑似笔误，需检查实际文件位置。
4. **摄像头索引**：`cv2.VideoCapture(0)` 固定使用索引 0，多摄像头环境需修改。
5. **Jupyter 内联显示性能**：`clear_output` + `plt.show()` 的实时刷新方案在 Notebook 中可行，但帧率较低，生产环境建议改用 `cv2.imshow()` 或 Web 方案。
6. **类别定义**：`data.yaml` 中的具体病虫害类别名称未在 Notebook 中体现，文档编写时需参考该文件。
