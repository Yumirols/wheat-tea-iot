## 需要解决的问题

- **问题描述**：图片存储持久化机制未定义 — 图片上传端点定义了返回路径，但 docker-compose 中无图片存储 volume 挂载，容器重建后历史图片将丢失
- **所在位置**：设计文档 §4.7.1 图片上传端点 + §5.2 docker-compose
- **严重程度**：一般
- **改进建议**：在 docker-compose 中增加图片存储 volume 挂载（如 `- ./images:/app/images`），或在 §4.7 端点说明中补充存储策略（本地磁盘/外部对象存储）

- **问题描述**：IoTDA 命令应答 Webhook 回传路径不完整 — 流 B 时序图显示命令应答通过 Webhook 回传 Python API，但 §4.2 仅定义了传感器和AI识别两条 Webhook 规则，缺少命令应答的转发规则
- **所在位置**：设计文档 §3.2 流 B 时序图 + §4.2 Webhook 配置
- **严重程度**：一般
- **改进建议**：方案 A（推荐）：在 IoTDA 增加第三条转发规则 rule_cmd_response_forward，将命令应答通过 Webhook 转发至 Python API；方案 B：通过轮询 IoTDA 设备影子获取命令执行结果
