# 农眼卫士 — 系统模块设计任务

## 任务概述

对"农眼卫士 (FarmEye Guard)"物联网系统进行详细的系统模块设计，包括系统总体设计、功能设计、工程文件组织结构等。需要特别关注系统模块之间的数据交互，如 VPS 上 Python API 提供的接口规范。不需要对每个模块内部进行详细设计。

## 必包含模块

| 序号 | 模块 | 说明 |
|------|------|------|
| 1 | 嵌入式端 (Embedded MCU) | WS63 Hi3863 + LiteOS，传感器采集、边缘AI结果接收、MQTT上报、命令执行、本地报警联动 |
| 2 | Python 上位机 (Host Computer) | 现场PC端集中监控与操作面板，PySide6/PyQt6 |
| 3 | AI 模型识别 | 边缘端病虫害视觉感知，CNN推理，病虫害分类与严重度分级 |
| 4 | 华为云设备接入 IoTDA | 设备注册认证、MQTT Broker、数据流转规则引擎、命令下发网关 |
| 5 | 鸿蒙应用 (HarmonyOS App) | 移动端监控与远程控制终端，ArkTS + ArkUI |
| 6 | 金仓数据库 (KingbaseES) | 国产化关系型数据存储 |

## 设计要求

1. **系统总体设计**：端-云-台三层架构定义、部署拓扑
2. **功能设计**：各模块职责与边界清晰定义
3. **模块间数据交互**：通信协议、数据流、接口规范（重点关注 Python API 接口）
4. **工程文件组织结构**：仓库顶层结构、各模块目录规划
5. **设计决策说明**：关键设计选择的原因

## 质量要求

- 所有数据字段与 DATA_INVENTORY.md 保持一致
- 接口规范完整覆盖 IoTDA Webhook、REST API、命令下发等
- 数据库表结构完整，索引设计合理
- 工程目录结构可落地实施
