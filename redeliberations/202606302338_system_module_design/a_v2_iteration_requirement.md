## 五、迭代历史经验教训（本轮参考）

1. **跨文档一致性是阻塞性检查项**：规格书、数据清单、模块设计文档之间的不一致必须在修订中解决。
2. **引用文件的完整性在审查前自检**：C-03 暴露了对外部文档的依赖问题，本轮必须彻底消除。
3. **API 边界条件设计不可省略**：多设备响应格式、分页上限行为等边界条件虽非主线流程，但缺失时导致歧义。
4. **安全设计不应被完全推迟**：即使在初版中，暴露在公网的 API 至少应具备最小化认证（参考 C-02 的讨论）。
5. **已知缺陷应显式标注但不可豁免**：设计文档的 TODO 标记是良好实践，但应确保后续追踪。
6. **传感器量程与报警阈值的匹配应基于数据手册**：H-06 暴露了阈值与传感器实际特性脱节的问题。
