# 质量质询：对 b_v3_diag_v1.md 的审查

## 审查结论

CHALLENGED

---

## 判定理由摘要

审查报告 b_v3_diag_v1.md 中，F-01 / F-03 / F-04 三项发现均有充分的代码证据支撑且逻辑自洽。但 **F-02** 存在证据不足和事实表述不准确的问题，影响了审查报告整体的严谨性。

---

## 详细质询

### F-02 [中] 证据不足与表述失实

**原发现主张**：诊断报告未评估 `server_default` 修复对集成测试断言假设的影响，并举例 `test_devices_columns` 等测试"会检查列属性的精确值"。

**质询**：

1. **证据链缺失**：原审查报告未引用任何实际测试断言代码来支撑"断言不兼容"这一论断。经核查 `server\tests\integration\test_db_ddl.py` 全部 13 个测试用例，没有一个测试用例对 `server_default` 的值做字符串对比或 `repr()` 断言。所有列验证测试仅检查**列名集合**（`set(columns.keys()) == expected`），不检查列的默认值、类型精确参数等属性。

2. **举例对象不匹配**：原审查报告称"例如 `test_devices_columns` 等测试会检查列属性的精确值"。但 `test_devices_columns`（`test_db_ddl.py` 第 88-89 行）的断言仅为：
   ```python
   expected = {
       "id", "device_id", "device_name", "mac_addr",
       "ip_addr", "registered_at", "last_seen",
       "online", "created_at",
   }
   assert set(columns.keys()) == expected
   ```
   验证仅涉及**列名集合**，与 `server_default` 的字符串或 `text()` 返回值类型无关。该举例不能支撑"断言不兼容"的论点。

3. **未核查关联测试**：F-02 还提及"可能有测试依赖于 `server_default` 的字符串值"。但审查报告未对 `test_db_crud.py` 中 `test_online_default_false`（第 236-242 行）进行核查。该测试直接通过 ORM 构造 `Device(device_id="default_test")` 并断言 `device.online is False`——此断言依赖 Python 端 `default=False`，与 `server_default` 的修复完全无关。该测试在问题 1 修复后仍应正常通过，不存在 F-02 所预警的风险。

**结论**：F-02 的证据不充分，引用的测试用例与所述风险不匹配。该发现应被归为"无实际代码依据的潜在风险提示"而非"有证据支持的审查发现"。建议在修订版中删除 F-02，或将其改写为"建议修复后运行完整集成测试回归"的最佳实践提示（不列入发现列表）。

---

### F-01 严重程度评定的商榷

**原发现主张**：F-01 评为"高"严重程度——替代方案缺少捕获返回值的先决条件说明。

**质询**：

F-01 指出替代方案需要在 `create_snapshot()` 中捕获 `ensure_device_exists()` 的返回值，这一事实判断是正确的。但将其评为"高"严重程度值得商榷：

- 替代方案在诊断报告中被表述为"也可以"，是推荐方案之外的附加选项，而非主要修复路径
- `ensure_device_exists()` 的返回值已在 docstring 中注明"返回 Device 对象"（原代码第 70 行），有 Python 基础的修复者添加 `device =` 是一个自然操作
- 主要推荐方案（方案 B）不存在此先决条件问题（在 `ensure_device_exists()` 函数体内操作，`device` 变量已在作用域内）

建议将该发现严重程度调整为"低"（仅影响替代路径的提示性补充），或维持原有评级但明确说明其仅影响替代方案而非主要修复路径。此项不影响整体审查结论，列作观察项。

---

## 整体评估

- **证据充分性**：F-01 / F-03 / F-04 具备充分代码证据；F-02 缺乏证据且存在事实表述不准确
- **逻辑完整性**：除 F-02 外，其余发现之间无矛盾，改进建议与问题一致
- **覆盖完备性**：审查覆盖了问题定位精确度、修复指令具体度、方案论证充分度、验收条件清晰度、交叉影响分析等维度

鉴于存在一项证据不足的核心发现（F-02），整体判定为 **CHALLENGED**。
