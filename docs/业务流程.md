## 2. 环境数据采集流程（泳道图）

```mermaid
sequenceDiagram
    participant Sensor as 传感器集群
    participant MCU as 嵌入式MCU
    participant Cloud as 华为云 IoTDA
    participant API as Python FastAPI
    participant DB as KingbaseES
    participant Client as 客户端

    loop 5s采集周期
        Sensor->>MCU: 温湿度数据
        Sensor->>MCU: 光照数据
        Sensor->>MCU: CO2浓度数据
        Sensor->>MCU: 超声波距离数据
    end

    MCU->>MCU: 数据预处理
    MCU->>MCU: 土壤NPK公式推算
    MCU->>MCU: 报警状态位计算
    MCU->>MCU: MQTT消息封装

    loop 10s上报周期
        MCU->>Cloud: MQTT属性上报 (QoS 1)
        Note over MCU,Cloud: service_id: farmeye_env<br/>携带全部环境参数
    end

    Cloud->>Cloud: 触发规则引擎
    Cloud->>API: HTTP POST Webhook转发
    Note over Cloud,API: 规则匹配后实时推送

    API->>API: 解析设备属性数据
    API->>DB: 写入传感器快照表
    API->>DB: 更新设备在线状态
    API-->>Cloud: 返回200 OK

    loop 10s轮询
        Client->>API: 查询最新传感器数据
        API->>DB: 读取最新一条快照
        DB-->>API: 环境数据
        API-->>Client: JSON格式响应
        Client->>Client: 刷新仪表盘展示
    end
```

## 3. AI识别与防治决策流程

```mermaid
sequenceDiagram
    participant User as 农户/农技员
    participant Client as 上位机/鸿蒙App
    participant AI as AI推理模块
    participant MCU as 嵌入式MCU
    participant Cloud as 华为云 IoTDA
    participant API as Python FastAPI
    participant DB as KingbaseES
    participant IoT as IoTDA命令API

    User->>Client: 上传或拍摄作物叶片照片
    Client->>AI: 提交图片进行CNN推理

    AI->>AI: 图像预处理
    AI->>AI: CNN模型前向推理
    AI->>AI: 病虫害类别判断
    AI->>AI: 置信度评估

    alt 置信度过低
        AI-->>Client: 识别不明确，提示重新拍摄
    else 置信度达标
        AI->>AI: 严重度分级评估
        AI-->>MCU: 板内通信传递识别结果
    end

    MCU->>Cloud: MQTT事件上报
    Note over MCU,Cloud: service_id: farmeye_ai<br/>携带病虫害识别结果

    Cloud->>API: HTTP POST Webhook转发
    API->>DB: 写入病虫害记录表

    API->>DB: 查询最近环境数据
    DB-->>API: 温湿度/光照/CO2数据

    API->>API: 决策引擎联动分析
    Note over API: 匹配病虫害类型×严重度×环境条件<br/>计算联动风险等级<br/>生成防治建议

    API->>DB: 更新联动分析结果

    alt 重度病虫害
        API->>DB: 写入控制日志
        API->>IoT: 调用IoTDA命令下发API
        IoT->>Cloud: 下发spray ON命令
        Cloud->>MCU: MQTT命令下发
        MCU->>MCU: 继电器吸合，启动喷淋
        MCU->>Cloud: MQTT命令应答
        Cloud->>API: Webhook命令应答转发
        API->>DB: 更新控制日志执行结果
    end
```

## 4. 手动控制设备流程

```mermaid
sequenceDiagram
    participant User as 农户/农技员
    participant Client as 上位机/鸿蒙App
    participant API as Python FastAPI
    participant DB as KingbaseES
    participant Cloud as 华为云 IoTDA
    participant MCU as 嵌入式MCU

    User->>Client: 在控制面板点击操作按钮

    alt 开启喷淋
        Client->>API: POST /api/v1/command<br/>{command: "spray ON"}
    else 关闭喷淋
        Client->>API: POST /api/v1/command<br/>{command: "spray OFF"}
    else 开启蜂鸣器
        Client->>API: POST /api/v1/command<br/>{command: "beep ON"}
    else 关闭LED
        Client->>API: POST /api/v1/command<br/>{command: "led OFF"}
    end

    API->>API: 参数校验
    API->>API: 检查设备在线状态

    alt 设备离线
        API-->>Client: 返回错误码1003
    else 设备在线
        API->>API: 生成命令唯一ID
        API->>DB: 写入控制日志
        API->>Cloud: 调用IoTDA命令下发API

        Cloud->>MCU: MQTT命令下发
        MCU->>MCU: 执行控制动作
        MCU->>Cloud: MQTT命令应答

        Cloud->>API: Webhook命令应答转发
        API->>DB: 更新控制日志执行结果
        API-->>Client: 返回命令下发成功
        Client-->>User: 界面显示控制结果
    end
```

## 6. 核心流程说明

### 6.1 环境监测流程
```
传感器定时采集 → MCU数据预处理 → NPK推导计算 → 
报警状态判断 → MQTT封装上报 → IoTDA规则转发 → 
API接收写入DB → 客户端轮询展示
```

### 6.2 AI识别与防治流程
```
图像采集 → CNN推理 → 病虫害分类 → 置信度评估 → 
严重度分级 → MQTT事件上报 → IoTDA转发 → 
决策引擎联动分析 → 防治建议生成 → 重度自动控制
```

### 6.3 报警控制联动
```
报警触发源：
├─ 环境参数超限（温湿度/光照/CO2/NPK）
└─ AI识别病虫害（按严重度分级）

联动执行：
├─ 轻度 → 界面提示 + LED绿色
├─ 中度 → LED黄色闪烁 + 蜂鸣器间歇 + 语音建议
└─ 重度 → LED红色闪烁 + 蜂鸣持续 + 语音紧急 + 自动喷淋
```

### 6.4 热力图生成逻辑
```
积累病虫害检测记录 → 按时间段/设备聚合 → 
统计频次与严重度分布 → 颜色映射渲染 → 
客户端展示交互式热力图
```

## 7. 异常流程处理

| 场景 | 处理方式 |
|------|---------|
| 传感器数据异常 | 标记异常值，跳过该次采集，显示"数据异常" |
| AI置信度过低 | 提示"识别不明确，建议重新拍摄" |
| MQTT连接断开 | MCU本地缓存数据，网络恢复后自动补报 |
| 设备离线 | 30秒无上报→标记离线，命令下发返回离线错误 |
| IoTDA推送重试 | API数据写入采用幂等机制防止重复记录 |
| 数据库写入失败 | 返回HTTP 500，触发IoTDA重试 |
| 命令下发失败 | 记录失败状态，返回错误码 |
| 数据导出超限 | 单次导出上限10万条，超限提示缩小范围 |
