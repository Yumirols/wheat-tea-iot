## 1. 项目背景

### 1.1 项目概述

我国是农业大国，小麦和茶叶是两大重要经济作物。传统病虫害识别依赖人工经验，存在效率低、误判率高、专家资源稀缺等问题。本项目"农眼卫士"旨在利用物联网环境监测技术、边缘端CNN图像识别技术和华为云IoT平台，构建**端-云-台三层架构**的智能病虫害识别与防治决策系统。

系统采用WS63 Hi3863嵌入式MCU作为现场控制核心，搭载温湿度、光照、CO₂、超声波等多种传感器进行环境数据采集，通过CNN模型实现病虫害智能识别与分级，结合华为云IoTDA进行设备管理与数据流转，由Python FastAPI后端提供业务逻辑与决策引擎，最终通过PySide6上位机和鸿蒙App实现多端交互。

### 1.2 项目目标

| 编号 | 目标 | 说明 |
|:----:|:----:|:------|
| G01 | 环境实时监测 | 实时采集并展示温湿度、光照、CO₂、土壤NPK、超声波距离等环境参数 |
| G02 | 病虫害智能识别 | CNN识别小麦锈病、白粉病、茶炭疽病、茶小绿叶蝉、蚜虫、红蜘蛛 |
| G03 | 严重程度分级 | 输出轻度/中度/重度三级，联动防治建议 |
| G04 | 环境-病害联动分析 | 结合环境数据评估病虫害扩散风险，生成防治决策 |
| G05 | 多级报警联动 | 轻度提示/中度LED蜂鸣/重度LED蜂鸣语音+自动喷淋 |
| G06 | 远程设备控制 | 支持自动触发和手动远程两种控制模式 |
| G07 | 云端数据持久化 | 传感器数据、病虫害记录、控制日志持久化存储与查询 |
| G08 | 多端可视化 | 上位机大屏仪表盘+鸿蒙App移动端，10s刷新实时数据 |

### 1.3 系统架构

系统采用**"端-云-台"三层架构**：

- **端（设备层）**：嵌入式MCU WS63 Hi3863 + LiteOS，搭载DHT11/LDR/MH-Z19C/HC-SR04传感器、LED/蜂鸣器/语音模块/继电器等执行机构，以及AI推理模块
- **云（服务层）**：华为云IoTDA（设备连接与消息流转）+ 境外VPS Docker环境（Python FastAPI + KingbaseES数据库），提供业务逻辑、数据持久化与决策引擎
- **台（表现层）**：PySide6上位机（PC端监控大屏）+ 鸿蒙App ArkUI（移动端远程控制），通过HTTP REST API交互

```
端（设备层） ──MQTT──▶ 华为云IoTDA ──Webhook──▶ 云服务层（FastAPI+DB）
                                                        │
              ◀──HTTP REST────────────────────────────┘
                                                        │
              台（表现层：上位机/鸿蒙App）◀──HTTP REST────┘
```

### 6.2 MQTT 通信规范

| 项目 | 规范 |
|:----:|:----:|
| MQTT版本 | 3.1.1 |
| 连接方式 | 保持长连接 |
| 属性上报方式 | 支持环境数据周期性上报和AI结果事件触发上报 |

**MQTT Topic约定**：
- 属性上报：设备→云
- 命令下发：云→设备
- 命令应答：设备→云

**service_id约定**：
- farmeye_env：传感器环境数据
